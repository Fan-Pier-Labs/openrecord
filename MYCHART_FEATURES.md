# MyChart Features Inventory

Comprehensive inventory of all MyChart features available through Epic's patient portal, compared against what this project currently scrapes.

> Generated by exploring a live MyChart session (March 2026).

## Legend

- **Have Scraper** — We have a working scraper for this
- **No Scraper** — Feature exists in MyChart but we don't scrape it yet
- **N/A** — Not useful to scrape (settings, UI-only features)

## No Scraper — High Value (11 features)

These are clinically or financially important data that would be valuable to scrape.

### 1. Medications
- **Menu Path:** My Record > Medications
- **URL:** `/Clinical/Medications`
- **API:** `api/medications/LoadMedicationsPage`
- **Data Available:**
  - Current medication list (name, common name, dosage instructions)
  - Prescription details (date prescribed, approving provider)
  - Refill details (quantity, day supply)
  - Pharmacy details (name, address, phone)
  - Renewal/refill request capability
- **Priority:** HIGH — Core health data

### 2. Immunizations
- **Menu Path:** My Record > Immunizations
- **URL:** `/Clinical/Immunizations`
- **API:** `api/immunizations/LoadImmunizations`
- **Data Available:**
  - List of all immunizations on file
  - Vaccine name, type, and all dates administered
  - Detailed view per immunization
- **Priority:** HIGH — Important for health records portability

### 3. Allergies
- **Menu Path:** My Record > Allergies
- **URL:** `/Clinical/Allergies`
- **API:** `api/allergies/LoadAllergies`
- **Data Available:**
  - Known allergies list (or "No known allergies")
  - Allergy type, reaction, severity
- **Priority:** HIGH — Critical safety data

### 4. Health Issues (Diagnoses / Problem List)
- **Menu Path:** (via Health Summary sidebar)
- **URL:** `/Clinical/HealthIssues`
- **API:** `api/HealthIssues/LoadHealthIssuesData`
- **Data Available:**
  - Current active health issues/diagnoses (e.g., "Major depressive disorder in full remission", "Myopia of both eyes")
  - Historical/resolved conditions
- **Priority:** HIGH — Core clinical data

### 5. Health Summary (Aggregate View)
- **Menu Path:** My Record > Health Summary
- **URL:** `/app/health-summary`
- **API:** `api/health-summary/FetchHealthSummary`, `api/health-summary/FetchH2GHeader`
- **Data Available:**
  - Height, weight, age (vitals)
  - Last visit info
  - Aggregated view of: medications, test results, health issues, immunizations, allergies
  - Recommended actions (overdue screenings, vaccines)
  - Health goals
- **Priority:** HIGH — Single endpoint that aggregates many data types, plus unique vitals data

### 6. Insurance Summary
- **Menu Path:** Insurance > Insurance Summary
- **URL:** `/Insurance`
- **API:** Insurance API (lazy-loaded)
- **Data Available:**
  - Insurance coverages on file
  - Plan details, subscriber info
  - Ability to add/update insurance card photos
- **Priority:** MEDIUM — Useful for billing/coverage tracking

### 7. Care Team
- **Menu Path:** Find Care > View Care Team
- **URL:** `/Clinical/CareTeam`
- **API:** CareTeam API (lazy-loaded)
- **Data Available:**
  - Primary Care Provider (name, specialty)
  - All care team members (specialists, NPs, etc.)
  - Provider photos, departments
  - Ability to message providers directly
- **Priority:** MEDIUM — Useful for provider directory

### 8. Preventive Care (Health Advisories)
- **Menu Path:** My Record > Preventive Care
- **URL:** `/HealthAdvisories`
- **API:** HealthAdvisories API
- **Data Available:**
  - Overdue screenings and procedures (e.g., "Cholesterol tests is overdue")
  - Recommended immunizations
  - Age/gender/history-based recommendations
- **Priority:** MEDIUM — Actionable health reminders

### 9. Referrals
- **Menu Path:** Find Care > Referrals
- **URL:** `/Referrals` → `/app/referrals`
- **API:** Referrals API (React app)
- **Data Available:**
  - Active and past referrals
  - Referring provider, specialty, status
- **Priority:** MEDIUM — Useful for tracking specialist referrals

### 10. Medical & Family History
- **Menu Path:** My Record > Medical and Family History
- **URL:** `/MedicalHistory` → `/app/histories`
- **API:** Histories API (React app)
- **Data Available:**
  - Patient medical history
  - Family medical history
  - Surgical history
  - Social history
- **Priority:** MEDIUM — Important background health data

### 11. Letters
- **Menu Path:** Communication > Letters
- **URL:** `/app/letters`
- **API:** Letters API (React app)
- **Data Available:**
  - Letters from providers (past 10 years, up to 100)
  - Downloadable/printable letters
- **Priority:** MEDIUM — Provider communications outside the messaging system

## No Scraper — Lower Value (8 features)

These are useful but less critical for data extraction.

### 12. Document Center
- **URL:** `/Documents`
- **Sub-sections:**
  - View/Download/Send Visit Records (`/app/record-download`)
  - Requested Records (`/Documents/Released`)
  - My Documents (`/app/document-center/my-documents`) — includes message attachments
- **Priority:** LOW-MEDIUM — Useful for downloading visit summaries and documents

### 13. COVID-19 Status
- **URL:** `/CovidStatus`
- **Data:** COVID-19 specific information, test results, vaccination records
- **Priority:** LOW — Overlaps with immunizations and test results

### 14. Track My Health
- **URL:** `/TrackMyHealth` → `/app/track-my-health/list`
- **Data:** Patient-entered health metrics, vitals tracking over time
- **Priority:** LOW — Patient-entered data, not provider data

### 15. To Do
- **URL:** `/todo`
- **Data:** Pending patient tasks (questionnaires, forms, etc.)
- **Priority:** LOW — Actionable items, not historical data

### 16. Upcoming Tests and Procedures
- **URL:** `/app/upcoming-orders`
- **Data:** Scheduled future lab orders, imaging, procedures
- **Priority:** LOW-MEDIUM — Useful for tracking upcoming care

### 17. Care Journeys
- **URL:** `/app/care-journeys`
- **Data:** Structured care plans and progress tracking
- **Priority:** LOW — Not all patients have active care journeys

### 18. Questionnaires
- **URL:** `/Questionnaires`
- **Data:** Pending and completed patient questionnaires
- **Priority:** LOW — Forms/surveys

### 19. Estimates
- **URL:** `/Estimates`
- **Data:** Cost estimates for upcoming procedures
- **Priority:** LOW — Not historical data

## Key API Endpoints Discovered

All endpoints are POST requests to `https://<hostname>/<firstPathPart>/api/...`:

| Endpoint | Returns |
|----------|---------|
| `api/health-summary/FetchHealthSummary` | Aggregated health summary (vitals, issues, meds, etc.) |
| `api/health-summary/FetchH2GHeader` | Health summary header (height, weight, age) |
| `api/medications/LoadMedicationsPage` | Current medications list |
| `api/immunizations/LoadImmunizations` | Immunization records |
| `api/allergies/LoadAllergies` | Allergy list |
| `api/HealthIssues/LoadHealthIssuesData` | Active/resolved health issues |
| `api/test-results/GetWidgetList` | Test results summary |
| `api/goals/LoadCareTeamGoals` | Care team health goals |
| `api/goals/LoadPatientGoals` | Patient-set health goals |
| `api/item-feed/FetchItemFeed` | Activity/notification feed |
| `api/education/GetPatEducationTitles` | Patient education materials |
| `api/personalInformation/GetContactInformation` | Contact info |
| `api/personalInformation/GetDetailsAboutMeInformation` | Demographics |
| `api/personalInformation/GetRelationships` | Emergency contacts, relationships |
| `api/personalInformation/GetContextIds` | Context IDs for patient |
| `api/central-jump-card/FetchCentralLink` | MyChart Central linking |

## Recommended Scraping Priority Order

1. **Medications** — Most universally useful health data
2. **Allergies** — Critical safety data, simple to scrape
3. **Health Issues** — Diagnoses/problem list
4. **Immunizations** — Vaccine records
5. **Health Summary** — Vitals + aggregate endpoint
6. **Care Team** — Provider directory
7. **Insurance Summary** — Coverage info
8. **Preventive Care** — Overdue screening alerts
9. **Medical & Family History** — Background health data
10. **Referrals** — Specialist tracking
11. **Letters** — Provider letters
